Oasis

Introduction

As project01 of the Web Development Immersive Program at GA, the goal of this project was to create an app that would allow users to search for and keep track of events in the area near them.

Initially this extended to the ability to scrape social media networks such as Facebook for events within the area, however this feature was soon differed for later versions due to time constraints.

The Solution

The project uses a simple Rails crud system to handle the creation of users and their relationships to events. Each user has a timeline from which they can search for events and add them to their schedule.

The timeline automatically takes note of the date of the event and slots it neatly between any events the user may already ahve queued up, this interaction is done mainly through Javascript, jQuery and AJAX calls.

On top of this, users are only able to search for events within a radius of their specific location, this was achieved through the manipulation of a geolocation gem that takes IP addresses and translates them to real physical locations.

The Code Base

  • HTML
  • SCSS
  • Ruby on Rails
  • Ruby Geocoder
  • PostGreSQL
  • Javascript
  • jQuery
  • GIT

Next steps

Further iterations of this app will include the following:

  • Improved UI
  • More refined and accurate IP geocoding via the use of cookies
  • Facebook OpenGraph API integration for event scraping
  • Abstraction of the timeline script into a jQuery plugin for integration with other third-party apps.

Drawsome

Introduction

This project allowed us to work in teams using git and collaborate on a larger project. The goal with this project was to create a real-time drawing game using an HTML canvas and websockets.

Although there were many of us across the project, everyone had significant exposure to all aspects of the design and implementation

The Solution

The game incorporates paper.js and websocket-rails to create a live drawing platform that supports multiple users seamlessly. Players are able to submit multiple guesses, with point scores decreasing linearly as the time approaches 0.

Game logic and requests are handled by the serverside (ruby on rails), while the MV* framework Backbone.js was used to render and manipulate views and collections on the clientside.

The game is also playable on both Mobile and Tablet devices with UI designed and developed to account for responsive use.

Prototypr: aka Bootleg Illustrator

Prototypr

The Problem

After reading an incredibly insightful article on the process behind the making of In Pieces, I decided to try and improve upon the process slightly by creating a web application for the dynamic creation of svg graphics within the browser.

The Solution

Incorporating snapSVG to aid in the manipulation of SVG elements in the DOM, the solution ended up being a rudimentary version of illustrator in the browser. The app allows users to upload an image as a reference behind the main SVG canvas.

Users can then plot points and create complex polygons using the Pen Tool, drag svgs around and edit their fill and stroke colors using the selection tool. The app also supports svg layering, users have the ability to change the order of svgs to achieve different layer effects just like in Adobe Illustrator.

Improvements

Future versions of this application will include:

  • Undo and redo functions for quickly making changes to the canvas
  • Support for HTML Canvas and CCSS Clip paths
  • Ability to export works easily as either svg files or pngs, or as a text file with relevant CSS styles or Canvas coordinates
  • Cloud storage of reference images and vector works for subscribed users
